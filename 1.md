# TÀI LIỆU YÊU CẦU DỰ ÁN (SRS) — ĐỀ TÀI 6: PHÂN TÍCH VĂN BẢN

> Phiên bản hoàn thiện (v2). Tài liệu đã được chuẩn hóa lại bố cục, bổ sung yêu cầu
> phi chức năng, quy ước mã hóa và hướng dẫn cài đặt/chạy chương trình.

## 1. Tổng quan đề tài
* **Tên đề tài:** Phân tích văn bản (Text Analyzer).
* **Mục tiêu:** Rèn luyện xử lý chuỗi nâng cao và tư duy phân tích dữ liệu cơ bản
  trên một ứng dụng desktop có giao diện Dark Theme.
* **Tính năng cốt lõi bắt buộc:**
    * Đếm số từ, số ký tự, số câu.
    * Tìm từ xuất hiện nhiều nhất.
    * Chuẩn hóa chuỗi (lowercase, loại bỏ dấu câu / punctuation).
    * Hiển thị Top N từ xuất hiện phổ biến nhất.

---

## 2. Kiến trúc hệ thống & Phân chia Module
Dự án được tổ chức theo kiến trúc phân lớp (Layered Architecture) để dễ bảo trì và
mở rộng. Tầng UI chỉ gọi xuống tầng Core/Utils; tầng Core **không** import ngược UI.

```text
project_root/
│
├── main.py                 # Điểm khởi động + Controller điều phối UI và Core Logic
├── config.py               # Cấu hình hệ thống (màu sắc, font, N mặc định, regex)
├── requirements.txt        # Khai báo thư viện phụ thuộc ngoài
│
├── core/                   # Tầng xử lý logic nghiệp vụ (Business Logic Layer)
│   ├── __init__.py
│   ├── preprocessor.py     # Chuẩn hóa chuỗi (lowercase, remove punctuation)
│   └── analyzer.py         # Đếm ký tự/từ/câu, trích xuất Top N từ phổ biến
│
├── ui/                     # Tầng giao diện (Presentation Layer)
│   ├── __init__.py
│   ├── main_window.py      # Khung giao diện chính (Dark Theme, Sidebar layout)
│   └── components.py       # Widget tái sử dụng (StatCard, OptionPanel, ...)
│
└── utils/                  # Tầng tiện ích hỗ trợ (Utility Layer)
    ├── __init__.py
    └── file_handler.py     # Đọc file (.txt, .docx) và xuất báo cáo (.txt)
```

## 3. Quy trình & Logic thực thi tối ưu
Luồng xử lý được tối ưu để chạy tốt với văn bản lớn:

1. **Nhập liệu (O(1)):** Nhận văn bản đầu vào từ Khu vực 1 (gõ trực tiếp, paste hoặc
   đọc từ file qua `utils/file_handler.py`).
2. **Đếm số liệu thô (O(N)):** Đếm số câu (dựa trên dấu `.`, `?`, `!`) và số ký tự
   trên **văn bản gốc trước khi chuẩn hóa** để giữ độ chính xác.
3. **Chuẩn hóa văn bản (O(N)):**
    * Dùng Regular Expression (`re`) để loại bỏ dấu câu.
    * Chuyển toàn bộ chuỗi sang chữ thường bằng `.lower()`.
4. **Phân tách từ & Thống kê tần suất (O(N)):**
    * Dùng `.split()` để tách văn bản thành danh sách các từ.
    * Dùng `collections.Counter` (thư viện chuẩn, tối ưu bằng C) để đếm tần suất với
      độ phức tạp tuyến tính $O(N)$. **Tuyệt đối không** dùng vòng lặp lồng nhau $O(N^2)$.
    * Gọi `.most_common(N)` để lấy nhanh Top $N$ từ phổ biến nhất.

## 4. Thiết kế Giao diện (UI/UX) & Cấu trúc chức năng

### 4.1. Chi tiết các khu vực chức năng
* **Khu vực 1 (Màn hình Nhập):** Ô nhập liệu lớn (Text Area). Hỗ trợ nhập trực tiếp,
  paste hoặc hiển thị nội dung trích xuất từ file `.txt` / `.docx`.
* **Khu vực 2 (Màn hình Xuất):** Ô hiển thị kết quả văn bản chỉ đọc (Read-only).
  Hiển thị toàn bộ văn bản sau khi chạy qua bộ lọc chuẩn hóa. Phía dưới là các
  Card/Bảng hiển thị số liệu thống kê (số từ, số ký tự, số câu, Top $N$).
* **Khu vực 3 (Bảng điều khiển tùy chọn):** Các Checkbox/Toggle để bật/tắt tính năng:
    * `[x]` Chuyển thành chữ thường.
    * `[x]` Xóa bỏ toàn bộ dấu câu.
    * `[x]` Thống kê số lượng (Từ, ký tự, câu).
    * `[x]` Tìm Top N từ xuất hiện nhiều nhất (kèm ô nhập số nguyên $N$).
* **Khu vực 4 (Bộ điều hướng hành động):** 2 nút bấm chính:
    * Nút `Tải File`: Mở hộp thoại chọn file `.txt` hoặc `.docx`.
    * Nút `Thực Thi`: Kích hoạt Controller xử lý các tính năng đã tích chọn ở Khu vực 3,
      sau đó đẩy dữ liệu hiển thị ra Khu vực 2.
    * (Bổ sung) Nút `Xuất báo cáo` và `Xóa` để tăng tính sử dụng.

### 4.2. Bố cục được chọn để triển khai — Biến thể 2: Cột chức năng (Sidebar Control)
* **Sidebar (cột trái — 25% chiều rộng):** Khu vực 4 (nút Tải file / Thực thi / Xuất /
  Xóa đặt trên cùng) và Khu vực 3 (danh sách Checkbox tùy chọn đặt phía dưới).
* **Khung hiển thị chính (cột phải — 75% chiều rộng):** Chia 2 nửa trên/dưới.
    * Nửa trên: Khu vực 1 (văn bản gốc).
    * Nửa dưới: hệ thống Tab — `Tab 1: Văn bản đã chuẩn hóa` (Khu vực 2),
      `Tab 2: Thống kê & Biểu đồ` (các StatCard + biểu đồ cột Top N).

> Biến thể 1 (Split View 50/50) vẫn là phương án dự phòng tương đương; mã nguồn tách
> layout khỏi logic nên có thể đổi bố cục mà không sửa tầng Core.

## 5. Yêu cầu phi chức năng (bổ sung)
* **Hiệu năng:** Toàn bộ pipeline phân tích phải đạt $O(N)$; xử lý file ~1MB mượt.
* **Robustness:** Văn bản rỗng, không có dấu câu, hoặc N lớn hơn số từ phải xử lý an
  toàn (không crash). N không hợp lệ → quay về `DEFAULT_N`.
* **Khả chuyển:** Chạy trên Windows/Linux/macOS có Python 3.10+.
* **Tách biệt tầng:** `core/` thuần Python, không phụ thuộc thư viện UI → dễ viết unit test.
* **Mã hóa:** Đọc/ghi file UTF-8 để hỗ trợ tiếng Việt có dấu.

## 6. Thư viện phụ thuộc (requirements.txt)
* `customtkinter` — xây dựng giao diện Dark Theme đồng bộ.
* `python-docx` — đọc văn bản từ file `.docx` trong `file_handler.py`.
* `matplotlib` — vẽ biểu đồ cột tần suất Top N (Tab 2).
